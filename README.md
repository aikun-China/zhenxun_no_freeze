## 📖 简介

`zhenxun_no_freeze` 是专为 **真寻Bot (zhenxun_bot)** 设计的进程守护工具。通过监控日志文件更新时间判断 Bot 是否僵死（无响应），自动执行清理并重启，确保 Bot 7×24 小时稳定运行。

---

## ✨ 功能特性

- 🔍 **日志监控** — 检测日志文件最后更新时间，判断 Bot 是否僵死
- 🧟 **僵死识别** — 超过阈值未更新日志即判定为僵死
- 🧹 **安全清理** — 精确识别项目进程，避免误杀其他程序
- 🔄 **自动重启** — 清理完成后自动启动 Bot
- 📜 **完整日志** — 记录每一步操作，便于排查问题
- 🛡️ **防误杀** — 通过可执行文件路径验证，确保只关闭属于项目的进程

## ⚙️ 配置说明

在 `watchdog_freeze.py` 顶部修改以下常量：

```python
# ========== 用户配置区 ==========

BOT_DIR = r"D:\桌面\qqai\zhenxun_bot-2026"   # 真寻Bot 主目录路径
START_BAT = os.path.join(BOT_DIR, "启动与管理.bat")  # Bot 启动脚本路径
LOG_DIR = os.path.join(BOT_DIR, "log")                # 日志监控目录
LOG_PATTERN = "*.log"                                   # 日志文件匹配模式

CHECK_INTERVAL = 300      # 检测间隔（秒），默认 5 分钟
FREEZE_THRESHOLD = 1800   # 僵死判定阈值（秒），默认 30 分钟

# ========== 以下一般无需修改 ==========
WATCHDOG_LOG = os.path.join(BOT_DIR, "data", "watchdog_freeze.log")  # 守护日志路径
PROJECT_MARKER = "zhenxun_bot-2026"  # 项目标识，用于进程识别
```

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `BOT_DIR` | — | 真寻Bot 主目录的**绝对路径** |
| `START_BAT` | `BOT_DIR\启动与管理.bat` | Bot 启动脚本路径 |
| `LOG_DIR` | `BOT_DIR\log` | 监控的日志目录 |
| `LOG_PATTERN` | `*.log` | 日志文件通配符 |
| `CHECK_INTERVAL` | `300` | 检测间隔（秒） |
| `FREEZE_THRESHOLD` | `1800` | 僵死判定阈值（秒） |

## 🚀 快速开始

### 1. 修改配置

打开 `watchdog_freeze.py`，将 `BOT_DIR` 修改为你的真寻Bot实际路径：

```python
BOT_DIR = r"D:\你的\实际\路径\zhenxun_bot-2026"
```

```bash
pip install psutil
```

> 注：代码中主要使用标准库（`os`, `time`, `glob`, `subprocess`, `ctypes`, `datetime`），`psutil` 为可选增强。

### 3. 启动守护

```bash
python watchdog_freeze.py
```

或使用 `.bat` 脚本包装后双击运行。

## 🔧 工作原理

```
┌─────────────────┐
│   启动守护程序   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐     否      ┌─────────────┐
│ 检测日志更新时间 │ ─────────► │ 判定正常    │
│  (LOG_DIR/*.log)│            │ 等待下一轮  │
└────────┬────────┘            └─────────────┘
         │ 是（超过阈值）
         ▼
┌─────────────────┐
│   判定为僵死    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  查找项目进程   │  ← 通过 PROJECT_MARKER 匹配 exe 路径
│  (tasklist +    │
│   GetModuleFileNameExW)
└────────┬────────┘
         ▼
┌─────────────────┐
│  安全结束进程   │  ← 验证路径后 taskkill /f /t
│  关闭控制台窗口 │  ← 匹配窗口标题，二次验证路径
└────────┬────────┘
         ▼
┌─────────────────┐     失败    ┌─────────────┐
│  等待清理完成   │ ─────────► │ 跳过启动    │

└────────┬────────┘            └─────────────┘
         │ 成功
         ▼
┌─────────────────┐
│   启动 Bot     │  ← os.startfile → subprocess.Popen 双保险
│  (START_BAT)    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  验证新进程     │  ← 20秒后检测3次，确认启动成功
│  是否存活       │
└─────────────────┘
```

## 🛡️ 安全机制

### 1. 路径验证防误杀

关闭进程前，通过 `GetModuleFileNameExW` 获取进程完整路径，确认包含 `PROJECT_MARKER`（`zhenxun_bot-2026`）才执行关闭。

### 2. 窗口标题二次确认

关闭控制台窗口时，匹配以下标题：
- `真寻Bot 综合管理控制台`
- `选择 真寻Bot 综合管理控制台`
- `Watchdog`
- `选择 Watchdog`

匹配后再次验证 exe 路径，避免误杀其他同名窗口。

### 3. 双保险启动

启动 Bot 时：
1. 优先使用 `os.startfile()`
2. 失败则回退到 `subprocess.Popen(..., CREATE_NEW_CONSOLE)`

### 4. 启动验证

发送启动命令后等待 20 秒，连续检测 3 次确认新进程已出现。

## 🐛 常见问题

**Q: 守护程序提示"未找到 Bot 进程"？**  
A: 检查 `BOT_DIR` 路径是否正确，以及 `PROJECT_MARKER` 是否与实际目录名一致。

**Q: 僵死后没有自动重启？**  
A: 查看 `data/watchdog_freeze.log`，确认清理步骤是否完成。如果"30秒内仍有残留"，可能是进程无法强制结束，需手动检查。

**Q: 能否同时守护多个 Bot 实例？**  
A: 可以复制多份 `watchdog_freeze.py`，分别配置不同的 `BOT_DIR` 和 `WATCHDOG_LOG` 路径。

**Q: 日志目录不在 `log/` 文件夹怎么办？**  
A: 修改 `LOG_DIR` 指向实际日志存放位置。
